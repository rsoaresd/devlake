# AgentReady Plugin

## Overview

The AgentReady plugin collects and analyzes AI readiness assessments from a centralized submissions repository. It fetches structured assessment JSON files via the GitHub API, extracts findings with tiered pass/fail results, and calculates aggregate metrics. Three Grafana dashboards provide fleet-level, findings-level, and per-repo visibility.

Assessments are generated by [AgentReady](https://github.com/ambient-code/agentready), an open-source tool that evaluates git repositories for AI-assisted development readiness. AgentReady scores repositories across 13 evidence-based categories and produces certification levels (Platinum, Gold, Silver, Bronze). This plugin ingests those assessment outputs into DevLake for centralized tracking and trend analysis across your fleet.

### Submissions Repository Structure

The plugin reads assessment files from a GitHub repository organized by org and repo:

```
submissions/
  {org}/
    {repo}/
      2026-02-07T14-42-31-assessment.json
      2026-03-01T10-00-00-assessment.json
  another-org/
    another-repo/
      ...
```

Each `{org}/{repo}` folder becomes a **scope** — a trackable unit that can be independently collected and analyzed.

## Plugin Architecture

This is a **data source plugin** using the DsHelper framework:

- Has its own Connection → Scope → ScopeConfig hierarchy
- Implements `DataSourcePluginBlueprintV200` interface
- Configured via Connections (like GitHub, GitLab plugins)
- References an existing GitHub connection for API authentication

## Connection Model

An AgentReady connection represents access to a submissions repository:

| Field | Type | Description |
|-------|------|-------------|
| `githubConnectionId` | uint64 | ID of an existing GitHub connection for API authentication |
| `submissionsRepo` | string | GitHub full name of the submissions repo (e.g., `ambient-code/agentready`) |
| `submissionsPath` | string | Subdirectory containing `{org}/{repo}/*.json` (default: `submissions`) |
| `branch` | string | Git branch to read from (default: repo default branch) |

## Data Models

### Assessment

Stores assessment records fetched from the submissions repository:
- Repository identity (ID, name, provider, connectionId)
- Schema version and overall score
- Certification level
- Attributes assessed vs. total available
- Branch, commit hash, and assessment timestamp
- Duration and raw JSON payload

Primary key: `(id, connection_id)` where id = `{repoId}:{commitHash}`

### Finding

Individual findings extracted from each assessment:
- Attribute identity (ID, name, category)
- Tier (1=Essential, 2=Critical, 3=Important, 4=Advanced)
- Status (pass, fail, skipped, error)
- Score, measured value, and threshold
- Evidence, remediation summary, and remediation steps

Primary key: `(id, connection_id)` where id = `{assessmentId}:{attributeId}`

### Metric

Aggregated metrics computed per assessment:
- Pass, fail, and skip counts
- Per-tier pass rates (Tier 1 through Tier 4) as percentages
- Category scores (JSON map of category to average score)

Primary key: `(id, connection_id)` where id = `{repoId}:{timestamp}:{assessmentId}`

## Subtasks

The plugin pipeline has three stages:

1. **collectSubmissions**: Fetches the Git tree from the submissions repo, filters entries matching the scope's `{org}/{repo}`, downloads each assessment JSON via the Contents API, and stores raw JSON.

2. **extractAssessments**: Parses raw JSON into structured assessment and finding records. Findings with status `not_applicable` are excluded. Depends on `collectSubmissions`.

3. **calculateMetrics**: Aggregates findings per assessment into pass/fail/skip counts, tier pass rates, and category scores. Depends on `extractAssessments`.

## Usage

### Prerequisites

1. A GitHub connection configured in DevLake
2. A submissions repository with assessment JSON files

### Setup via Config-UI

1. Go to **Connections** → **AgentReady** → **Create Connection**
2. Select a GitHub connection, enter submissions repo info
3. Test the connection
4. Add scopes (discovered automatically from the submissions tree)

### Setup via API

```bash
# Create connection
curl -X POST http://localhost:4000/api/plugins/agentready/connections \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-assessments",
    "githubConnectionId": 1,
    "submissionsRepo": "ambient-code/agentready",
    "submissionsPath": "submissions"
  }'

# Discover scopes
curl http://localhost:4000/api/plugins/agentready/connections/1/remote-scopes

# Register scopes
curl -X PUT http://localhost:4000/api/plugins/agentready/connections/1/scopes \
  -H "Content-Type: application/json" \
  -d '[{"fullName": "myorg/myrepo", "name": "myrepo"}]'
```

### Blueprint Configuration

```json
{
  "plugin": "agentready",
  "options": {
    "connectionId": 1,
    "fullName": "myorg/myrepo"
  }
}
```

### CLI Debugging

```bash
cd backend
go run plugins/agentready/agentready.go -c 1 -f "myorg/myrepo"
```

## Database Tables

- `_tool_agentready_connections`: Connection configuration
- `_tool_agentready_scopes`: Tracked `{org}/{repo}` scopes
- `_tool_agentready_scope_configs`: Per-scope configuration
- `_tool_agentready_assessments`: Assessment records with scores and metadata
- `_tool_agentready_findings`: Individual findings per assessment
- `_tool_agentready_metrics`: Aggregated pass rates and category scores

## Grafana Dashboards

### Fleet Overview (`fleet-overview.json`)
Aggregate view across all repositories: average scores, certification distribution, and fleet-level trends.

### Findings Analysis (`findings-analysis.json`)
Top 15 most common failures (bar chart) and tier pass rates over time (time series).

### Repository Detail (`repo-detail.json`)
Single-repo deep dive with current score, certification level, and detailed findings breakdown.

## Tier Model

| Tier | Name | Description |
|------|------|-------------|
| 1 | Essential | Baseline requirements every repository must meet |
| 2 | Critical | Important practices for production readiness |
| 3 | Important | Recommended practices for mature repositories |
| 4 | Advanced | Best-in-class practices for leading repositories |
